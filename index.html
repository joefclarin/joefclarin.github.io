CHALLENGES

This website probably the most complicated search functionality that I've built so far. The website itself has so many content and most of them are linked together in one way or another. If you go to the Reports page, you'll see lots of possible ways to filter the search. This is already beyond the normal capabilities of the WordPress query and any other ways to inject anything on that query.

The way to solve this is obviously creating a raw SQL queries but another challenge here is the content structure. Since most of the content are connected to each other, a number of nested SQL queries is a must. I spent a good amount of time into the WordPress database to understand how to properly build the raw SQL and at the same time not mess up with the content structure.

Challenges

I'm not sure if it's an Australian thing but property listings in Australia are stored in a big database and companies such as Agentbox offers an API to access this data. One of the challenges of this website is using Agentbox's API. It's a challenge in a way because their documentation lack the proper information because they prefer their clients to either acquire their website development business, AgentPoint, or use their partners that offers WordPress plugin and/or theme to build your website.

The workaround for this though is instead of using the API, Agentbox will send XML files to the server then script which is triggered by cronjob processess the XML files and store the information to WordPress database.

Challenges

The most complicated part on this website is the practitioner slider. It might not be noticeable at first but the main custom post types in this website are connected to each other in some way, and three of those are displayed in the practitioner slider. Since this slider is not part of the original requirement, the custom post types are already structured in a particular way so in order to make this slider work, the series of nested SQL queries are done.

Takeaways

This website itself is not very complicated for users but this ironically makes it complicated for developers like me. I like how the data are structured. This is actually the first time in a while that a client can understand data structure jargon which made me happy.

Although the practitioner slider almost made me crazy, I actually enjoy working on this website. It forced me to think beyond the usual WordPress query.
